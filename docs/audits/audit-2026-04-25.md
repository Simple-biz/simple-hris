## 1. Bug fixes

### 1.1 CSV source-picker treated oldest as "latest" (two places)

`/api/hubstaff-hours?source_files=1` returns files **newest-first**, but two consumers were defaulting to `files[files.length - 1]` (oldest) and labelling the dropdown by reversing the array — so the "(latest)" tag landed on the oldest entry.

| File | Old | New |
|---|---|---|
| `src/components/employee/EmployeeDashboard.tsx:387, 839` | `setSelectedFile(files[files.length - 1])` | `setSelectedFile(files[0])` |
| `src/components/employee/EmployeeDashboard.tsx:1493` | `[...sourceFiles].reverse().map((file, i) => …)` | `sourceFiles.map((file, i) => …)` |
| `src/components/Overview.tsx:1201` | `setSelectedSourceFile(files[files.length - 1])` | `setSelectedSourceFile(files[0])` |
| `src/components/Overview.tsx:1691` | `[...sourceFiles].reverse().map(…)` | `sourceFiles.map(…)` |

`(latest)` now sits on the first entry of the dropdown, which is also the default selection.

### 1.2 Next 16 dev server blocked LAN HMR

When loading the dev server from a second device on the LAN (e.g. `http://192.168.0.101:3000`), the page rendered but the websocket failed because Next 15+ blocks cross-origin dev assets unless the origin is whitelisted.

**Fix** — `next.config.ts`:
```ts
allowedDevOrigins: ["192.168.0.101"],
```

Also documented for the user that Google OAuth's redirect-URI policy disallows private IPs (`192.168.x.x`); to test SSO from another device they need a hostname (hosts-file alias or a tunnel like ngrok/cloudflared/Tailscale Funnel).

### 1.3 Add Employee modal — bottom clipped

The previous modal used `overflow-y-auto` on the body with an inline `style={{ maxHeight: "min(70vh, 560px)" }}`, then put the footer **outside** the scroll region. On certain viewport heights, the Compensation section would overflow the inline cap, push the footer down, and the submit button could be clipped.

**Fix** — modal completely rebuilt (see §3.4). New layout uses `flex h-[min(92vh,720px)] flex-col` so header, scrollable body, and footer divide the dialog into three regions; the submit button can never be clipped.

## 2. Feature: PAB department-scope removed

PAB previously had a per-department gate (`pab_scope_department_keys` setting + `isDeptInPabScope` helper). This added complexity that wasn't being used in practice and forced the employee dashboard to render an "out of scope" branch for everyone whose department wasn't in the picker. Per user direction (`2026-04-25`): **PAB now applies to every employee, no per-department gating.**

### 2.1 Code removed

| File | Removed |
|---|---|
| `src/lib/pab-period-settings.ts` | `PAB_SCOPE_DEPARTMENT_KEYS_KEY`, `parsePabScopeDepartmentKeys`, `isDeptInPabScope`, `scopeDepartmentKeys` field on `PabPeriodFetchResult`, fetch+parse step in `fetchPabPeriodSettings`. |
| `src/hooks/usePabPeriodSettings.ts` | `scopeDepartmentKeys: null` from initial state. |
| `src/components/employee/EmployeeDashboard.tsx` | `pabAppliesToEmployee` useMemo, `employeeDeptKey` state + setter calls, `'out_of_scope'` enum on `perfectAttendanceBonusStatus`, the "PAB not enabled for your department" warning card, all branches keyed on either of the above. Imports trimmed: `isDeptInPabScope`, `normalizeDeptToKey`. |
| `src/components/PayrollWizard.tsx` | `if (!isDeptInPabScope(...))` gate in the eligible-set computation, the `resolveDeptKey` helper that fed it, `pabPeriodSettings.scopeDepartmentKeys` from the deps array. Import of `isDeptInPabScope` dropped. |
| `src/components/Overview.tsx` | Same gate at the org-wide PAB metric, plus the now-unused `emailToDeptKey` map. Imports of `isDeptInPabScope` and `normalizeDeptToKey` dropped. |

### 2.2 Data left untouched

The stored row in `app_settings` keyed `pab_scope_department_keys` (if present) is now dormant — nothing reads or writes it. **Safe to delete by hand later but no migration is required.** The setting key was deliberately not blacklisted from inserts, so re-introducing the feature in the future would just need to re-add the consumers.

### 2.3 Behavior change

- Every employee sees the PAB calendar and the Monthly PAB / Tech Bonus stats on their Overview, regardless of department.
- The Payroll Wizard PAB calculator no longer filters by department — every Hubstaff row is evaluated.
- The org-wide "PAB eligible" count on the Accounting Overview no longer filters by scope.

## 4. Feature: My Disputes page (rebuilt)

The `disputes` tab was a placeholder card. It now hosts a working dispute management page.

### 4.1 New components

#### `src/components/employee/MyDisputes.tsx` *(new)*

Two-column layout at `lg+` (form left, history right; counter chips below the form). On mobile, calendar at the top, form, then history.

Form fields (state lives in `MyDisputes` only; no parent contract beyond `prefill`):
- `disputeDate` — `<input type="date" max={today}>`. Pre-filled when the parent passes `prefill`.
- `reason` — `<select>` defaulting to `orphanage_visit`. Options come from settings if `pab_dispute_reason_codes` is set, otherwise the `DEFAULT_DISPUTE_REASON_CODES` fallback (see §4.3).
- `explanation` — `<textarea>`. Required when `reason === 'other'`.

Submit calls `POST /api/pab-disputes` with `{ work_email, dispute_date, reason, explanation, created_by }` (same shape as the old `DisputeDialog`). On success the form resets, `calendarRefreshKey` is bumped to refetch disputes in the inline calendar, and the history list re-loads.

The page also accepts a `prefill?: { date: string; seconds?: number }` prop. When set, the form auto-fills the date, scrolls into view, and notifies the parent via `onPrefillConsumed` so the parent can clear its prefill state.

#### `src/components/employee/EmployeePabCalendar.tsx` *(new)*

Self-contained PAB calendar, used inside MyDisputes. Mirrors the visual + interaction model of the calendar inside `EmployeeDashboard` but does its own data fetching (sources → merged row → calendar weeks → disputes) so it doesn't drag in the dashboard's file-picker / payroll state.

Props:
- `employeeEmail: string`
- `onCellClick?: (payload: { date: string; seconds: number; dispute: PabDayDisputeRow | null }) => void`
- `refreshKey?: number` — bump to force a disputes-only refetch (e.g. after a successful submit).
- `className?: string`

Click model:
- Past sub-7h days, with or without an existing dispute → fires `onCellClick`.
- Today / future days, or days that meet 7h on their own → ignored.

### 4.2 Dashboard PAB calendar → navigation, not modal

Old: clicking a sub-7h cell in the dashboard PAB calendar opened the inline `<DisputeDialog>`.
New: the dashboard fires `onNavigateToDisputes({ date, seconds })`, the parent `EmployeeApp` flips the active tab to `disputes` and seeds `disputesPrefill`, and `MyDisputes` consumes it.

| File | Change |
|---|---|
| `src/components/employee/EmployeeDashboard.tsx` | New `onNavigateToDisputes?: (prefill?: { date: string; seconds?: number }) => void` prop. Cell `onClick` now calls it instead of opening a modal. Stripped: `disputeDialogOpen`, `selectedDisputeDay`, `selectedExistingDispute` state, `DisputeDialog` import, the dialog JSX. |
| `src/components/employee/EmployeeApp.tsx` | Added `disputesPrefill` state, plus a `<MyDisputes>` render in the `disputes` case (was a placeholder). The `dashboard` case now wires `onNavigateToDisputes` so calendar clicks land on the disputes form. |

`src/components/employee/DisputeDialog.tsx` is now an unused file — left in place to minimise blast radius; safe to delete in a follow-up.

## 6. Feature: Payroll Wizard "Upload CSV" tab — uniform grid

The three upload kinds (master list / payroll rates / Hubstaff timesheet) used to stack vertically with three different visual shapes (two were colored cards, the Hubstaff one was a plain heading + horizontal toolbar). They now share the same card shape and sit in a responsive grid.

Layout:
- **Mobile** (`<md`): 1 column.

Color accents preserved per type — emerald (roster), sky (rates), indigo (Hubstaff). The "Hogan cycle" toggle moved inside the Hubstaff card, right above its upload button.

Removed: the "Hubstaff timesheets" `<Separator>` divider that used to sit between the 2nd and 3rd cards (now redundant with per-card grouping).
